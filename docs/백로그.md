## 백로그
**백엔드: https://github.com/kxb2/backend**
**프론트엔드: https://github.com/kxb2/frontend**

1. INPUT (프론트 채민기님)
- 9컷 생성 버튼
> 생성 버튼을 클릭하면 그리드 생성 진행
> 조건에 맞지 않을 시 버튼 비활성화

- 이미지 모델 선택
> 사용자가 이미지를 출력할 AI 모델을 선택(GPT Image, Gemini 3.5 Falsh Image)
> 선택한 모델은 전체에 일괄 적용(컷별 혼용 x)
> 재생성 시 모델 변경 X

- 세부 설정
> 그림체(스타일), 톤, 화면비, 시대 배경을 설정할 수 있는 설정 제공(디테일 요소 추가)

- 시나리오 입력
> 시나리오 입력 텍스트 박스 제작
> 9컷 생성 버튼 제작(클릭 시 그리드 및 샷별 프롬프트 통합 생성)
> 추상 무드어(역동적, 다양한 등) 사용 금지(모델이 평균값으로 뭉갬)
> 한국어 입력
> 시나리오 텍스트를 입력하지 않으면 버튼 비활성화(필수)

- 장르 선택
> 장르 필터 추가(드라마, 액션, 로맨스, 스릴러, 코미디)(장르는 변경 가능성이 존재)
> 앵글 아크 프리셋 제공(샷 타입 자동 배정, 참고 PRD 7번)
> 클릭 시 결과에 반영

- 이미지/동영상 업로드
> 이미지/동영상 업로드 공간 제작
> 이미지를 업로드 할 경우에는 최소 1장 ~ 최대 10장까지
(이미지 업로드를 하지 않은 경우에도 생성은 가능)


2. COMPONENTS


3. EXPORT (백엔드 담당)
- 내보내기 결과 확인 (2순위):
내보내기 완료 여부와 다운로그 링크를 확인할 수 있어야 한다

- 이미지 내보내기 (1순위):
3*3 그리드 이미지 1장을 기본으로 내보낸다 (추후 옵션으로 개별 이미지도 내보낼 수 있도록 한다)

- PDF 내보내기 요청 (1순위):
9컷의 이미지와 프롬프트가 들어간 pdf를 만들어 다운로드 링크를 반환한다


4. OUTPUT - AI (프론트 채민기님)
- 컷별 재생성
> 재생성을 통해 해당되는 컷만 재생성(전체가 재생성되면 안됨)

- 통합 프롬프트
> 샷별 프롬프트를 묶어 통합된 프롬프트로 제공
> 통합된 프롬프트에선 샷별 구분이 있어야함
ex) Shot 1, Shot 2, etc.
> 영어 출력 고정(품질 최상)
> 합계 3000자 이내(실시간 카운터)
> 3000자 초과 시 시각적 경고 + Export 차단

- 샷별 프롬프트
> 해당되는 샷에 대한 설명 제공
> 앵글 명시 강제(앵글명 + 카메라 위치 + 결과 묘사)
ex) low-angle, 카메라가 바닥에서 올려다봄, 인물이 위압적으로 보임
> 추상 무드어 출력 금지
> 영어 출력 고정(품질 최상)
> 행동은 현재형 동사 1개, 단일 동작. 단일샷 50~70단어 가이드
> 장르 프리셋 + 고급 설정값 + 시나리오를 기본 틀로 제공

- 그리드(3x3)
> Input의 내용을 기반으로 3x3 그리드 이미지를 생성
> 컷 순서 고정, 컷별 재생성 지원이 필요


5. OUTPUT - CANVAAS (프론트 이유진님)
- 캔버스 툴바:
캔버스 결과물/기능 및 도구의 팔레트 역할을 하는 툴바

- 생성 결과물 매핑:
생성된 이미지와 프롬프트 매핑을 그대로 표시(읽기 전용), 캔버스가 매핑을 새로 생성하거나 수정하지 않음

- 이미지 Export:
생성된 이미지 9컷을 3x3 그리드 형식의 이미지로 내보내기 (컷 개별 분할 출력 기능은 선택 사항), 파일 형식/해상도는 확인 필요

- PDF Export:
생성된 이미지 9컷 및 프롬프트(샷별로 분해 표기)를 한 문서로 내보내기 (샷 번호 + 프롬프트가 시각적으로 명확하게 매핑되어야 함)

- 생성 결과 버전 관리:
생성된 이미지와 프롬프트의 버전을 저장 및 선택하여 불러올 수 있게 함

- 생성 결과 메모:
생성된 이미지와 프롬프트 주변에 메모를 작성할 수 있게 함

- 생성 결과 노드 연결:
생성된 이미지와 프롬프트가 캔버스 내에서 선으로 연결될 수 있게 함

- 생성 결과물 자유 배치:
생성된 이미지와 프롬프트를 자유롭게 표시 배치/정리하는 기능

- 캔버스 UI:
생성된 9컷·프롬프트를 배치하는 정리용 작업 공간 (스토리보드 생성 기능과 독립된 별도 기능)


6. 스토리보드 (백엔드 담당)
- 스토리보드 조회 (1순위):
저장된 스토리보드의 정보 (시나리오/장르/이미지)를 불러온다

- 레퍼런스 이미지 업로드 (1순위):
캐릭터/배경/소품 등의 이미지를 업로드 받아 저장하고 저장된 이미지를 AI에게 반환한다

- 장르 프리셋 기본 값 자동 적용 (3순위):
고급 설정이 없을 경우 선택한 장르의 기본 값(톤,스타일)이 자동으로 채워져야 한다

- 스토리보드 생성 (1순위):
시나리오 텍스트, 장르, 이미지를 받아 저장한다 (추후 고급 설정도 입력 받을 수 있음)


7. 캔버스 (백엔드 담당)
- 캔버스 조회 (2순위):
저장된 컷 배치 위치를 불러올 수 있어야 한다

- 캔버스 배치 저장 (2순위):
각 컷의 위치(X,Y 좌표로) 저장한다


8. 컷 생성 (백엔드 담당)
- 컷 재생성 (2순위):
9컷 중 실패하거나 마음에 안드는 컷이 있을 경우 재생성한다 (이때 컷 구별을 위해 컷ID 부여해야함)

- 컷별 프롬프트 적용 (1순위):
각 컷 별로 프롬프트가 연결되도록 묶는다

- API 연동 (1순위):
GPT,Gemini,claude api를 연동한다

- 컷 생성 상태 확인 (2순위):
이미지 생성의 진행사항을 주기적으로 확인 가능하게 한다(대기/생성/완료/실패 로 분류)

- 9컷 생성 요청 (1순위):
생성 버튼을 누르면 AI에게 스토리보드 값을 전달한다 (9컷 생성 병렬처리 가능하다면 병렬처리 할거임)
