## 1. 목적 / 한 줄 정의

> 🎯 **Objective (검증 목표)** — 시나리오·장르·키 이미지(선택) 입력만으로 9컷 스토리보드와 컷별 영상 제작 프롬프트를 자동 생성하고, 그 산출물이 영상 제작 AI에 바로 투입 가능한 수준인지 검증한다.
> 
> 
> **Hypothesis (검증 가능한 가설)** — 장르 프리셋과 자동 프롬프트 조립을 제공하면, 프롬프트에 미숙한 일반 사용자도 별도 학습 없이 영상 제작 AI 투입 가능 수준의 스토리보드·컷별 프롬프트를 한 번에 생성한다.
> 

**핵심 정의**

- 기획(텍스트)과 영상화 사이의 스토리보드 단계를 자동화한다.
- 입력 — 시나리오 프롬프트 + (선택) 복수 레퍼런스 이미지.
- 출력 — 9컷 아웃컷(**3×3 그리드 1장**) + 샷별 영문 프롬프트(**통합 1개**). 다음 단계(영상 AI)로 전달.
- 키 이미지(레퍼런스)는 **선택 입력** — 미첨부 시 텍스트만으로 9컷 생성, 첨부 시 참조용 최대 10장(0~10장), 로컬 수동 업로드.

**전체 흐름상 위치**

기획(텍스트) → **[B] 스토리보드 9컷(그리드) → 통합 프롬프트** → 영상 AI(첫 프레임 정밀 생성 → 15초 영상화) → 합치기 → 음악·자막

**제품 컨셉**

- "전문가(방송 PD)가 만든, 누구나 쉬운 도구" · 실사용 타깃 10~80대 · 기준 사용성 30~40대 · 접근성 최우선.

> 💡 **캔버스(F-04)는 생성 파이프라인의 단계가 아니다.** 독립된 자유 배치 보드로 §3·§5 F-04에서 별도 정의한다.

## 2. 성공 기준 (데모에서 증명할 것)

**데모 완주 기준**

- 한국어 시나리오 1건 입력 → **9컷 그리드 + 통합 영문 프롬프트** → Export까지 끊김 없이 1회 통과.
- 목표는 완성도가 아니라 파이프라인 완주. 텍스트 기반·이미지 첨부 두 경로 모두 성립해야 한다.

**캔버스 성공 기준 (독립·비차단)**

- 빈 보드에서 임의 이미지/영상을 올려 자유 배치·연결·메모 가능. 완주 기준과 독립적으로 별도 성립한다.

> ⚠️ **KPI·이벤트 스키마·A/B 실험 설계는 본 범위에서 제외한다.** 본 프로젝트는 지표로 가설을 통계 검증하는 실험형이 아니라 파이프라인을 끝까지 통과시키는 데모 완주형 MVP다. (PRD 작성요령 7·8절 대비 예외 — 근거를 명시해 규율 유지)

## 3. User Flow

**생성 파이프라인 (스토리보드) — 단일 흐름**

입력 → 생성 → 검수·재생성 → 프롬프트 확인 → Export → 영상 AI 영상 제작

| 단계 | 사용자 행동 | 대응 기능 |
| --- | --- | --- |
| 1 | 시나리오 입력 · 장르 선택 | F-01 |
| 2 | (선택) 고급 설정 조정 | F-01 고급 |
| 3 | (선택) 레퍼런스 이미지 첨부(0~10장) · 생성 실행 | F-01 / F-02 |
| 4 | 9컷(3×3 그리드) 검수 | F-02 |
| 5 | 특정 컷 재생성 | F-02 컷별 재생성 |
| 6 | 샷별 프롬프트 · 글자수 확인 | F-03 |
| 7 | PDF / 이미지 Export | F-05 / F-06 |
| 8 | 생성 프롬프트·이미지로 영상 AI 영상 제작 | 범위 경계 |

작업물 관리 — 스토리보드·캔버스 작업물은 라이브러리(목록)에서 조회·재진입한다(F-08).

**캔버스 (독립 기능 · 파이프라인 외)**

- 빈 캔버스에 임의 이미지/영상 반입 → 섹션·커넥터·메모로 자유 배치·정리(F-04, 병렬·비차단).

> 💡 **다운스트림(영상 AI)** — 3×3 그리드 1장 + 통합 프롬프트로 15초 통짜 영상 1개 생성. Seedance 2.5 대응은 미개발(관찰만, §10).

## 5. 기능 명세 + 수용 기준

### F-01 시나리오 입력 (P0) — 기본/고급 2단 구조

**기본 화면 (항상 노출)**

- 시나리오 텍스트박스 1개(한국어 자연어·자유 입력).
- 장르 칩 1줄 — 드라마 / 액션 / 로맨스 / 스릴러 / 코미디(5개 내외).
- 레퍼런스 이미지 첨부 — **선택 입력** · 최대 10장(0~10장).
- [9컷 생성] 버튼.

> ✅ **입력 검증 규칙(확정)** — 0장 → 생성 버튼 활성(텍스트 기반) · 1~10장 → 활성(참조 기반) · 10장 초과 → 첨부 차단 + 안내 노출.
> 

**고급 설정 (접힘·펼침)**

- 그림체(스타일) · 톤 · 화면비 · 시대 배경.
- **화면비 기본값 = 3:2 고정** — 이미지 모델이 비율 3종(가로/세로/정사각)만 지원하므로 9컷 전부 가로 3:2 적용. 사용자 조절은 고급 옵션으로 향후 제공.
- 미조정 시 장르 프리셋 기본값 적용, 조정 시 오버라이드.

원칙 — 좋은 기본값이 선택지를 대체한다. 사용자 80%는 시나리오+장르로 완료하며, 프롬프트 구조화는 시스템이 책임진다.

### F-02 9컷 생성 (P0) — 3×3 그리드 1장 + 모델 비종속

> ✅ **생성 경로 2분기(확정)** — ① 이미지 있음(1~10장) → 참조 기반 생성 · ② 이미지 없음(0장) → 텍스트 기반 생성. 두 경로 모두 산출·핸드오프 단위 동일(3×3 그리드 1장).
> 
- 입력 — (선택) 레퍼런스 이미지(0~10장) + 시나리오 → **3×3 그리드 1장(9컷 통합 이미지)**. 검수는 컷 단위, 산출은 그리드 1장.
- 화면비 — 전 컷 3:2 고정.
- **컷별 재생성** — 1컷 오류 시 전체 재생성 없이 해당 컷만 재생성(품질 튜닝의 핵심). 재생성 결과는 그리드 해당 위치에 갱신.
- **컷 순서 고정** — 시나리오 내러티브와 연동. 순서 변경 시 컷↔프롬프트 매핑이 어긋난다. 스토리보드 내부 속성이며 캔버스(F-04)와 무관.
- 이미지 모델 어댑터 레이어 경유 — API 키 + 모델 문자열 교체로 모델 스왑. 프리뷰 모델 문자열 사용 금지(GA만).
- 장르 앵글 아크 프리셋에 따라 9컷 각각 샷 타입 자동 배정(컷별 수동 오버라이드 가능).
- 컷별 앵글 명시 강제 — [앵글명 + 카메라 물리 위치 + 결과 묘사] 3겹 구조. 추상 무드어 사용 금지.

**이미지 모델 선택**

- 사용자 선택형 2종 — GPT image(기본값) / Gemini. 9컷 전체 일괄 적용(컷별 혼용 불가), 어댑터 동일 인터페이스로 호출.

### F-03 샷별 프롬프트 (P0) — 영어 고정 + 통합 1블록 + 자동 조립

- 출력 영어 고정 — 한국어 입력을 시스템이 영어로 변환·생성.
- **통합 프롬프트 1개 블록** — 내부에 샷별(Shot 1~9) 구분. 영상 AI 투입 시 블록 통째 복사, PDF Export 시 샷번호+설명으로 분해 표기.
- Seedance 6파트 순서 적용 — Camera → Subject → Action → Environment → Lighting → Style.
- 시스템이 [장르 프리셋 + 고급 설정값 + 시나리오]를 받아 사용자가 구조를 몰라도 자동 조립.
- 행동은 현재형 동사 1개·단일 동작, 단일샷 50~70단어 가이드.
- **합계 3000자 실시간 카운터**(Seedance 입력 제약) + 백엔드 총합 검증. 초과 시 시각적 경고 + Export 차단/경고.
- **자체 프롬프트 가이드라인 삽입** — 팀 노하우(앵글/구도/품질)를 프롬프트 단계에 자동 삽입해 사용자 간 품질 편차를 보정.

### F-04 캔버스 (전체 필수·병렬) — 독립된 자유 배치 보드

정의 — 자유 배치형 작업 보드(피그마 하위 버전). 빈 캔버스에서 시작하며, 사용자가 이미지·영상·메모를 직접 반입·배치하고 요소 간 관계를 정리한다. **생성 기능 없음(정리 전용).** **1인 사용 전제.**

**툴바 구성 (메모로 통일)**

| 툴 | 용도 |
| --- | --- |
| 포인터 | 요소 선택 |
| 뷰어 포인터 | 화면 이동(패닝) |
| 섹션 | 그룹화(요소 묶기, 더블클릭 제외/재편입) |
| 메모 | 텍스트/프롬프트 입력(제목+내용) — 텍스트·댓글 대체·통일 · 노드 연결 가능 · 색상 커스텀 |
| 이미지·비디오 추가 | 임의 이미지/영상 반입(드래그앤드롭 다중/개별) · 영상=재생표시→클릭 재생 |
| 노드(커넥터) 연결 | 요소 간 선 잇기 — 사용자 직접 · 관계(흐름) 표시용, 작동/생성 없음 |
- **메모 통일 원칙** — 1인 사용 전제이므로 협업용 '댓글'과 별도 '텍스트' 툴을 두지 않고 '메모' 하나로 흡수(제목+내용). 프롬프트용 별도 메모 툴 없음.
- 코어 조작 — 자유 드래그·크기 조절·회전·삭제·undo/redo, 이미지 다중 추가(10장 제한).
- 범위 포함 — 코어(섹션·첨부·커넥터·메모·드래그) + 고급(자동 정렬·버전 관리) 모두 본 범위, 병렬 개발.

> ⚠️ **의존성 경계** — ① 독립 보드(산출물 비종속, 임의 반입 가능) · ② 역방향 쓰기 없음(캔버스 조작이 스토리보드 로직·컷 순서·매핑에 영향 없음) · ③ 재정렬/컷 순서 개념 없음 · ④ 비차단(건너뛰어도 생성·Export·핸드오프 성립). 노드그래프식 "생성형" 캔버스는 범위 밖(§8) — 커넥터는 관계 표시용이며 합성·생성 트리거가 아니다.
> 

### F-05 PDF Export (P0)

- 9컷 + 샷별 프롬프트를 한 문서로 출력 — 팀 공유용 레이아웃. 샷번호 + 설명 매핑 명확(공유용은 샷별 분해 표기).

### F-06 이미지 Export (P0)

- **3×3 그리드 1장** 출력(영상 AI 투입 기본 형식). 필요 시 컷 개별 분할 출력 옵션. 영상 AI 투입 가능한 형식/해상도.

### F-07 인증 / 회원가입 (P0)

- 보안 Lv2 진입 게이트 — 미인증 사용자는 스토리보드 생성·Export·캔버스 접근 불가.
- 로그인 방식 2종 — ① 구글 소셜 로그인 ② 이메일 회원가입/로그인.
- 이메일 회원가입은 **사내 도메인 제한 없이 오픈**(가입 링크 보유 시 누구나, PD 승인). 접근 통제는 로그인 후 리소스 권한 층위에서 유지.

### F-08 라이브러리 (목록) — P1

- 스토리보드 전체 목록 + 캔버스 전체 목록 조회. 사용자(로그인)별 저장분, 항목 선택 시 재진입.
- 진입 UI — 헤더 버튼 클릭 → 좌측 슬라이드 사이드바(오버레이, 뒷배경 흐림). 기존 좌측 배치를 유지하도록 오버레이 처리.

## 7. 장르 프리셋 매핑 (초안)

장르 선택 시 기본값을 일괄 적용한다. 9컷 앵글 아크는 "설정→전개→절정→여파" 흐름에 샷 타입을 매핑한 것이다. 톤·스타일·화면비는 고급 설정에서, 개별 컷 앵글은 F-02에서 오버라이드한다. 컷 순서 자체는 고정(내러티브 연동).

| 장르 | 기본 톤 | 기본 스타일 | 9컷 앵글 아크 (컷1 → 컷9) |
| --- | --- | --- | --- |
| 드라마 | 따뜻함·차분 | 실사/내추럴 | wide → medium → close-up↑ → close → two-shot → medium → close-up → wide → medium |
| 액션 | 긴장·강렬 | 실사/시네마틱 | extreme wide → low-angle medium → Dutch → tracking wide → close-up → high-angle → low-angle → wide → medium |
| 로맨스 | 부드러움·따뜻 | 소프트/감성 | wide → medium two-shot → close-up → over-the-shoulder → close-up → soft wide → two-shot → close → medium |
| 스릴러 | 어두움·불안 | 로우키/차가움 | wide(여백) → high-angle → Dutch → extreme close-up → low-angle → POV-feel → high-angle → wide → close |
| 코미디 | 밝음·경쾌 | 선명/하이키 | wide → medium → close-up → two-shot → wide → medium → close → high-angle → wide |

주의 — 이 표는 출발선이다. 실제 품질은 각 앵글을 영문 프롬프트로 풀 때 [물리 위치 + 결과 묘사]의 구체성에 좌우된다. 5개 장르는 시작점이며, B1/B2가 튜닝 후 좋은 아크를 취합한다.
