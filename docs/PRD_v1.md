## 1. 목적 / 한 줄 정의

기획(텍스트)과 영상화 사이의 스토리보드 단계를 자동화한다. 사용자가 입력한 시나리오 프롬프트와 **(선택) 복수 레퍼런스 이미지**를 받아 **9컷 아웃컷(3×3 그리드 1장) + 샷별 영문 프롬프트(통합 1개)** 를 생성하고, 다음 단계(영상 AI를 통한 영상화)로 넘긴다.

**키 이미지(레퍼런스)는 선택 입력이다.** 이미지를 넣지 않아도 시나리오·인물·배경 묘사 등 텍스트만으로 9컷 생성이 가능해야 한다. 넣을 경우 캐릭터·배경·부가 캐릭터·사물 등을 참조용으로 최대 10장까지 첨부할 수 있다(최소 0장 ~ 최대 10장). 레퍼런스 이미지는 사용자가 로컬에 저장한 이미지를 수동 업로드한다.

전체 흐름상 위치: 기획(텍스트) → [B] 스토리보드 9컷(그리드) → 통합 프롬프트 → 영상 AI(첫 프레임 정밀 생성 → 15초 영상화) → 합치기 → 음악·자막

**제품 컨셉:** “전문가(방송 PD)가 만든, 누구나 쉬운 도구”. 실사용 타깃은 10~80대 일반 사용자이며 기준 사용성은 30~40대가 쉽게 쓰는 수준(80대까지 가능하면 베스트). 접근성 최우선 — 단, 연령/카테고리 자동 분기와 초심자용 버튼식 버전은 본 범위 밖(향후 고도화, §8).

> **캔버스(F-04)는 위 생성 파이프라인의 단계가 아니다.** 스토리보드 생성과 독립된 자유 배치 보드로, §3·§5 F-04에서 자기 정의한다.
>

## 2. 성공 기준 (데모에서 증명할 것)

데모 목표는 “한국어 시나리오 1건 입력 → **9컷 그리드 + 통합 영문 프롬프트** → Export까지 끊김 없이” 한 번에 통과하는 것. 화려함보다 파이프라인 완주가 핵심이다. **완주 시나리오는 이미지 없이(텍스트 기반)와 이미지 첨부(참조 기반) 두 경로 모두 성립해야 한다.**

캔버스는 이 완주 기준과 독립적이다(비차단). 캔버스 성공 기준은 “빈 보드에서 임의 이미지/영상을 올려 자유롭게 배치·연결·메모할 수 있다”로 별도 성립한다.

**캔버스 (독립 기능 · 파이프라인 외):**

| 단계 | 사용자 행동 | 대응 |
| --- | --- | --- |
| (별도) | 빈 캔버스에서 임의 이미지/영상을 올려 프레임·노드 연결·메모로 자유 배치·정리 | F-04 (병렬·비차단) |

데모 뼈대는 “만들고(9컷 그리드) → 내보낸다(Export·핸드오프)”. 따라서 생성·출력을 P0 최상단에 둔다. **캔버스는 후순위가 아니라 스토리보드 빌더와 병렬로 개발한다.**

## 5. 기능 명세 + 수용 기준

### F-01 시나리오 입력 (P0) — 기본/고급 2단 구조

**기본 화면 (항상 보임)**

- 시나리오 텍스트박스 1개 (한국어 자연어, 자유 입력)
- 장르 칩 1줄 — 드라마 / 액션 / 로맨스 / 스릴러 / 코미디 (5개 내외)
- 레퍼런스 이미지 첨부 — **선택 입력.** 넣지 않아도 생성 가능(텍스트 기반). 넣을 경우 **최대 10장까지**(캐릭터·배경·부가 캐릭터·사물 등) 참조용으로 첨부. **허용 범위: 최소 0장 ~ 최대 10장.**
- 업스트림 자동 연계 없음, 유저가 준비한 이미지를 로컬 저장 후 수동 업로드
- [9컷 생성] 버튼

> **입력 검증 규칙(확정):** 첨부 이미지 수량은 0장 이상 10장 이하만 유효하다.
- 0장 → 생성 버튼 활성(텍스트 기반 생성 경로).
- 1~10장 → 생성 버튼 활성(참조 기반 생성 경로).
- 10장 초과 → 첨부 차단. “참조 이미지는 최대 10장까지 첨부할 수 있습니다” 안내 노출.
> 

→ 장르만 고르면 즉시 생성 가능(이미지 없이도). 나머지는 장르 프리셋 기본값이 자동 적용.

**고급 설정 (접힘, 펼쳐야 보임)**

- 그림체(스타일) · 톤 · 화면비 · 시대 배경
- 안 건드리면 장르 프리셋 기본값 적용. 건드리면 오버라이드
- ※ 비주얼 스타일(애니메이션/실사 등)은 이 고급 설정의 그림체·톤에 포함(별도 최상단 인풋 신설 없음). PD 회신 기준 우선 포함, 이후 필요 없으면 제거 검토(§10).

**원칙:** 좋은 기본값이 선택지를 대체한다. 80%는 시나리오+장르로 끝, 디테일 원하는 20%만 고급 설정을 펼침. 사용자가 프롬프트 순서·형식을 직접 구조화하지 않는다(구조화는 시스템 책임).

### F-02 9컷 생성 (P0) — 3×3 그리드 1장 + 모델 비종속 + 장르 앵글 아크

> **생성 경로 2분기(확정):**
- **① 이미지 있음(1~10장) →** 참조 이미지 기반 생성. 업로드한 캐릭터·배경·부가 캐릭터·사물을 참조해 9컷 구성.
- **② 이미지 없음(0장) →** 시나리오 + 인물·배경 묘사 등 텍스트 기반 생성. 참조 없이 시나리오 해석만으로 9컷 구성.
- 두 경로 모두 산출·핸드오프 단위는 동일(3×3 그리드 1장). 첨부 상한은 10장이며 초과 입력은 F-01에서 차단된다.
> 
- (선택) 복수 레퍼런스 이미지(0~10장) + 시나리오 → **3×3 그리드 1장(9컷 통합 이미지)** 생성. 검수는 컷 단위로 확인 가능하되 산출·핸드오프 단위는 그리드 1장(영상 AI 투입 형태와 일치)
- 컷별 재생성 지원 — 1컷만 틀려도 전체를 다시 안 돌려도 됨 (품질 튜닝의 핵심). 재생성 결과는 그리드의 해당 위치에 갱신
- **컷 순서 고정** — 9컷 순서는 시나리오 내러티브와 연동되어 순서를 바꾸면 컷↔︎프롬프트 매핑이 꼬인다. 이는 **스토리보드(아웃컷) 내부 속성**이며, 캔버스(F-04)와는 무관하다(캔버스에는 컷 순서·재정렬 개념이 없다).
- 이미지 모델 어댑터 레이어 경유 — API 키 + 모델 문자열 교체만으로 모델 스왑
- 프리뷰 모델 문자열 사용 금지 — GA 버전만 사용(프리뷰는 예고 없이 종료됨)
- 선택된 장르의 앵글 아크 프리셋에 따라 9컷 각각에 샷 타입 자동 배정 (컷별 수동 오버라이드 가능)
- 컷별 앵글 명시 강제 — [앵글명 + 카메라 물리 위치 + 결과 묘사] 3겹 구조로 프롬프트화 (예: “low-angle, 카메라가 바닥에서 올려다봄, 인물이 위압적으로 보임”)
- 추상 무드어(“역동적”, “다양한”) 사용 금지 — 모델이 평균값으로 뭉갬

**이미지 모델 선택**

- 9컷 생성 시 사용자가 이미지 모델을 선택 (GPT image(상위 버전) / Gemini 3.5 Flash Image)
- 기본값: GPT image · 사용자가 Gemini 3.5 Flash Image로 변경 가능
- 선택한 모델은 9컷(그리드) 전체에 일괄 적용 (컷별 혼용 불가). 컷별 재생성도 현재 선택된 모델로만 수행
- 어댑터 레이어를 통해 동일 인터페이스로 호출

### F-03 샷별 프롬프트 (P0) — 영어 출력 고정 + 통합 1블록 + 자동 조립

- 출력은 영어 고정 (영상 AI·이미지 모델 모두 영어에서 품질 최상). 사용자 입력(한국어) → 시스템이 영어로 변환·생성
- **산출은 통합 프롬프트 1개 블록** — 내부에 샷별 설명(Shot 1·2·…·9)이 구분되어 들어감. 영상 AI 투입 시 블록 1개를 통째로 복사/드래그. PDF Export 시에는 샷번호+설명으로 분해 표기
- Seedance 6파트 순서 적용: Camera → Subject → Action → Environment → Lighting → Style
- 시스템이 [장르 프리셋 + 고급 설정값 + 시나리오]를 받아 사용자가 구조를 몰라도 위 순서로 자동 조립
- 행동은 현재형 동사 1개, 단일 동작. 단일샷 50~70단어 가이드
- 합계 3000자 실시간 카운터 (Seedance 입력 제약). 초과 시 시각적 경고 + Export 차단(또는 경고)
- **자체 프롬프트 가이드라인 삽입** — 외부 상용 엔진이라 모델 정밀조정 불가. 일반 사용자의 프롬프트 미숙을 보완하기 위해 팀 노하우(앵글/구도/품질 가이드라인)를 프롬프트 단계에 자동 삽입해 품질 편차를 줄인다

### F-04 캔버스 (필수 ) — 독립된 자유 배치 보드

**정의:** 자유 배치형 작업 보드(피그마 하위 버전). **빈 캔버스에서 시작**하며, 사용자가 이미지·영상·텍스트를 **직접 올려** 원하는 위치에 자유롭게 배치하고 요소 간 관계를 정리한다. **생성 기능은 없다(정리 전용).** 스토리보드 아웃컷을 올려 참고 관계를 정리할 수도 있으나, 아웃컷에 한정되지 않고 임의 이미지/영상을 반입할 수 있다.

**코어 기능 (필수):**
- 프레임 생성
- 이미지·영상 첨부 (임의 반입 — 아웃컷 전용 아님)
- 노드 연결(요소 간 선 잇기) — 연결은 **사용자가 직접 수행**(관계 표시용)
- 프롬프트/메모 텍스트 공간
- 요소 **자유 드래그 배치**(원하는 위치로 이동)

**구성:** 좌측 ’컷 대기열’은 두지 않는다(임의 이미지/영상 반입으로 대체). 빈 캔버스에서 사용자가 반입·정리한다.

**범위 제외:** 자동 정렬·버전 관리 등 고급 기능은 향후(§8, 일정 여유 시).

> **의존성 경계 (스토리보드 ↔︎ 캔버스, 결합 오해 방지):**
- **독립 보드.** 캔버스는 스토리보드 산출물에 종속되지 않는다. 아웃컷을 올릴 수도 있으나 반드시 그럴 필요는 없다(임의 이미지/영상 반입).
- **역방향 쓰기 없음(no write-back).** 캔버스 조작은 스토리보드 생성 로직·컷 생성 순서·컷↔︎프롬프트 매핑을 바꾸지 않는다.
- **재정렬/컷 순서 개념 없음.** 캔버스는 요소를 자유 배치할 뿐, ’9컷을 재정렬’하는 기능이 아니다(컷 순서는 F-02 스토리보드 내부 속성).
- **비차단(non-blocking).** 캔버스를 건너뛰어도 스토리보드 생성·Export·영상 AI 핸드오프는 그대로 성립한다.
- ⚠️ 노드그래프식 **“생성형” 캔버스**(이미지를 선으로 연결해 합성·생성)는 초기 올인원 원안의 잔재이며 본 범위 밖(§8). 캔버스의 노드 연결은 사용자가 **관계를 표시**하는 용도이지 합성·생성 트리거가 아니다.
>

## 7. 장르 프리셋 매핑 (초안)

장르를 고르면 아래 기본값이 한 번에 깔린다. 9컷 앵글 아크는 “설정 → 전개 → 절정 → 여파” 흐름에 샷 타입을 매핑한 것. 톤·스타일·화면비는 고급 설정에서, 개별 컷 앵글은 F-02에서 오버라이드 가능. 컷 순서 자체는 고정(내러티브 연동).

| 장르 | 기본 톤 | 기본 스타일 | 9컷 앵글 아크 (컷1 → 컷9) |
| --- | --- | --- | --- |
| 드라마 | 따뜻함, 차분 | 실사/내추럴 | wide(설정) → medium → close-up↑ → close → two-shot → medium → close-up → wide → medium |
| 액션 | 긴장, 강렬 | 실사/시네마틱 | extreme wide → low-angle medium → Dutch angle → tracking wide → close-up → high-angle → low-angle → wide → medium |
| 로맨스 | 부드러움, 따뜻 | 소프트/감성 | wide → medium two-shot → close-up → over-the-shoulder → close-up → soft wide → two-shot → close → medium |
| 스릴러 | 어두움, 불안 | 로우키/차가움 | wide(여백) → high-angle → Dutch angle → extreme close-up → low-angle → POV-feel → high-angle → wide → close |
| 코미디 | 밝음, 경쾌 | 선명/하이키 | wide → medium → close-up → two-shot → wide → medium → close → high-angle → wide |

**주의:** 이 표는 출발선이다. 실제 품질은 각 앵글을 영문 프롬프트로 풀 때 [물리 위치 + 결과 묘사]를 얼마나 구체적으로 쓰느냐에 달려 있다. 5개 장르는 시작점이며 B1/B2가 튜닝 후 좋은 아크를 취합한다.

## 10. 오픈 이슈 / 확인 필요

**해소 (확정):**

- ✅ **입력값(키 이미지 정책)** — 키 이미지=선택 입력 확정. 최소 0장 ~ 최대 10장 첨부. 0장은 텍스트 기반, 1장 이상은 참조 기반 생성. 참조용 이미지는 캐릭터·배경·부가 캐릭터·사물 등. 업스트림 자동 연계 없음(수동 업로드).
- ✅ NFR 수준 — 성능 Lv2 / 보안 Lv2 / 품질 Lv1 (+ 프롬프트 가이드라인 보정)
- ✅ 디자인 레퍼런스/톤 — 4종 수령 + 네온 핑크-퍼플 톤·대형 입력창 방향 확정
- ✅ 백엔드 인력 — B1·B2 각 1명 배정 확정
- ✅ 모델 갱신 — 이미지 GPT image/Gemini 3.5 Flash, 프롬프트 Claude(비용표 반영)
